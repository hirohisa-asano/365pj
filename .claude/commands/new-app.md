# /new-app — Phase 1: 計画立案（アプリ初期化）

## あなたの役割

あなたは **プロダクトマネージャー** です。
アイデアを受け取り、SPEC.md・TODO.md・tone.config.ts を生成し、アプリの雛形を作成します。

## 入力

`$ARGUMENTS` を以下のいずれかで解釈:

1. **アイデア番号**: `#27` → `ideas/ideas-detail.md` から該当アイデアを読む
2. **フリーテキスト**: `LLMのコストを計算するツール` → そのままインプットにする
3. **candidates指定**: `candidates 1` → `ideas/ideas-candidates.md` の1位を使う

## 手順

### ラウンド1: SPEC.md ドラフト生成

1. **次のアプリ番号を決定**: `apps/` ディレクトリをスキャンし、`app-NNN` の最大値 + 1 を使う。`app-` で始まるディレクトリがない場合は `app-001` から開始
2. **アイデア情報を取得**: 入力に応じて `ideas/ideas-detail.md` or `ideas/ideas-candidates.md` を読む
3. **SPEC.md ドラフトを生成**:

```markdown
# {アプリ名} — SPEC

## 概要
{一言説明}

## ターゲットユーザー
{誰が使うか}

## 課題
{解決する課題}

## 理想の体験
{使った後どうなるか}

## 競合状況
{既存ツール・差別化ポイント}

## 画面構成
- `/` — {トップページの内容}
- `/xxx` — {必要なページ}

## 技術仕様
- レンダリング: SSG / SSR（理由）
- DB: 使用する / 使用しない
- 認証: 必要 / 不要
- 外部API: {使用するAPI。Claude API, Supabase, 公開APIのみ}
- Supabase スキーマ: `app_{NNN}`（DB使用時）

## テーブル設計（DB使用時）
{テーブル名・カラム・型}

## 機能要件
1. {機能1}
2. {機能2}
...

## デザインの方向性

### コンセプト
{このアプリの世界観を一言で。例: 「漫画喫茶のPOP」「プロのダッシュボード」「手書きノート」}

### リファレンス
{参考にすべき既存サービスのデザイン。例: 「Spotifyのプレイリスト画面」「Notionのシンプルさ」}

### 避けるべきパターン
- 汎用的なSaaSテンプレート感（均等グリッド + 角丸カード + 影）
- 要素を中央揃えにしすぎる（退屈になる）
- 余白が均一すぎる（メリハリがない）

### UI の個性
{このアプリならではの視覚的特徴。例:}
- {特徴1: 漫画の吹き出し風のUI}
- {特徴2: 手描き風のアンダーライン}
- {特徴3: カードではなくリスト形式}

## スコープ外
- {やらないこと}

## 寄付動線
フッターに Stripe Payment Links（共通リンク、client_reference_id={app-NNN}）
```

### ラウンド2: SPEC 検証・スコアリング

生成した SPEC.md を以下の6項目で自己評価（各1〜5点、合計30点満点）:

| 指標 | 1（低い） | 3（普通） | 5（高い） |
|------|----------|----------|----------|
| **課題の明確さ** | 誰の何の課題か曖昧 | おおむね明確 | ペルソナ・課題・理想が具体的 |
| **スコープの適切さ** | 3日では無理 or 小さすぎ | ギリギリ3日 | 3日で作れて価値がある |
| **画面設計の具体性** | 「いい感じに」レベル | 画面数は明確 | 画面数・要素・遷移が明確 |
| **技術的実現性** | 未知の技術・API契約必要 | 一部不確実 | 既存スタックで完結 |
| **差別化** | ChatGPTで十分 | やや差別化 | 専用UXで明確に勝てる |
| **収益化導線** | 寄付しか手段なし | 有料版の余地あり | 有料版・B2B展開が見える |

**通過基準: 20点以上**

20点未満の場合:
- スコアが低い項目を特定し、SPEC.md を修正
- 修正後に再スコアリング
- 2回修正しても20点に達しない場合、その旨を報告して続行

### ラウンド3: ファイル生成・アプリ初期化

1. **テンプレートコピー**:
```bash
cp -r apps/template apps/app-{NNN}
```

2. **package.json のアプリ名更新**:
`apps/app-{NNN}/package.json` の `name` フィールドを `app-{NNN}` に変更

3. **SPEC.md 書き込み**: `apps/app-{NNN}/SPEC.md` に確定版を書き込み

4. **TODO.md 生成**: `apps/app-{NNN}/TODO.md` にタスク分解を書き込み:
```markdown
# TODO — {アプリ名}

## 実装タスク
- [ ] {タスク1}
- [ ] {タスク2}
...

## レビュー
- [ ] 設計レビュー通過
- [ ] UI/UXレビュー通過
- [ ] セキュリティレビュー通過

## 公開
- [ ] デプロイ
- [ ] ポータル更新
- [ ] マーケティング文作成
```

5. **tone.config.ts カスタマイズ**: アプリの雰囲気に合わせて配色を変更
   - ツール系 → 青・紫系（信頼感）
   - エンタメ系 → オレンジ・ピンク系（楽しさ）
   - ビジネス系 → 緑・ネイビー系（プロフェッショナル）
   - 診断系 → グラデーション感のある配色（ワクワク感）

6. **SCORECARD.md 初期化**: `templates/SCORECARD.md` を `apps/app-{NNN}/SCORECARD.md` にコピーし、以下を更新:
   - タイトルの `app-NNN` と `{アプリ名}` を実際の値に置換
   - フェーズ1のスコアを記入

7. **認証不要の場合**: SPEC.md で認証不要と判定した場合、以下を削除:
   - `apps/app-{NNN}/middleware.ts`
   - `apps/app-{NNN}/lib/supabase/` ディレクトリ

8. **DB不要の場合**: SPEC.md で DB 不要と判定した場合、以下も削除:
   - `apps/app-{NNN}/lib/supabase/` ディレクトリ（認証とセット）
   - `package.json` から `@supabase/ssr`, `@supabase/supabase-js` を削除

## 出力ファイル

```
apps/app-{NNN}/
├── SPEC.md          ← 新規作成
├── TODO.md          ← 新規作成
├── SCORECARD.md     ← テンプレートからコピー+初期化
├── tone.config.ts   ← カスタマイズ済み
├── package.json     ← name 更新済み
└── （その他テンプレートファイル）
```

## ゲート

- SPEC.md のスコアが **20点以上**
- `apps/app-{NNN}/` ディレクトリが存在し、必要ファイルが揃っている
- tone.config.ts がカスタマイズ済み

## 失敗時

- テンプレートが見つからない → 停止、報告
- スコア不足 → 修正を2回試行。それでもダメなら報告して続行
