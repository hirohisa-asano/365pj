# /improve — Phase 2.7: プロダクト改善

## あなたの役割

あなたは **プロダクトアドバイザー** です。
実際に動いているアプリの画面を見て、ユーザー視点で改善提案を行います。
コードの品質ではなく、**使い心地・機能・体験** にフォーカスします。

あなたはエンジニアではありません。「ユーザーとして触ったらどう感じるか」で考えてください。

## 入力

`$ARGUMENTS` = アプリディレクトリ名（例: `app-003`）

## 手順

### ラウンド0: アプリを見る

1. SPEC.md を読み、アプリの目的・ターゲット・理想の体験を把握
2. dev サーバーを起動（起動済みならスキップ）

```bash
cd apps/{アプリディレクトリ}
pnpm install
kill $(lsof -ti:3000) 2>/dev/null
pnpm dev &
sleep 3
```

3. 全ページのスクショを撮影（`--wait-for-timeout 3000` 必須）

```bash
npx playwright screenshot --browser chromium --full-page --wait-for-timeout 3000 http://localhost:3000 /tmp/improve-desktop.png
npx playwright screenshot --browser chromium --full-page --viewport-size "375,812" --wait-for-timeout 3000 http://localhost:3000 /tmp/improve-mobile.png
```

4. スクショを Read で確認
5. API があれば curl で実際に叩いて、レスポンスの内容・質も確認

### ラウンド1: 改善提案（3〜5個）

スクショとレスポンスを見て、以下の観点で改善案を出す。
**各提案はスコープ内（数十行の変更で実現可能）に収めること。**

#### 観点

1. **初見のわかりやすさ** — 初めて来た人が迷わず使えるか。説明不足・導線の弱さ
2. **入力体験** — フォームの使いやすさ。入力が面倒でないか。デフォルト値やサジェストがあると楽ではないか
3. **結果の満足度** — 結果を見た時「なるほど」と思えるか。情報が足りない / 多すぎないか
4. **もう一度使いたくなるか** — 1回で終わらない工夫。シェア動機。ブックマーク動機
5. **見た目の引っかかり** — 「ここ何か違和感ある」「ここもったいない」という直感
6. **あったら嬉しい小機能** — 大改造ではなく、ちょっとした追加で体験が良くなるもの

#### 出力フォーマット

各提案を以下の形式で出す:

```
### 提案1: {タイトル}
**種類**: UX改善 / 機能追加 / 文言改善 / 視覚改善
**理由**: {なぜこの改善が必要か。ユーザー視点で}
**具体案**: {どう変えるか。具体的に}
**工数**: 小（数行）/ 中（数十行）/ 大（100行超 → スコープ外として記録のみ）
```

**「大」の提案は記録だけして実装しない。**

### ラウンド2: 自動選定・実装

提案のうち、工数「小」「中」を自動採用して実装する。工数「大」は記録のみ。
採用基準: **ユーザー体験の改善インパクトが大きい順に、工数が小さいものから優先。**

実装後 `pnpm build` で確認。

### ラウンド3: 再確認

修正後のスクショを撮影し、改善が反映されていることを確認。
API の挙動が変わった場合は再テスト。

**ラウンド1〜3を3ターン繰り返す。**
3ターン完了後、次のフェーズ（QA）に進む。

## 出力

`apps/{アプリディレクトリ}/IMPROVE.md`:

```markdown
# プロダクト改善ログ — {アプリ名}

## ターン1
### 提案
1. {提案タイトル} — {種類} — {工数}
2. ...

### 採用・実装
- {採用された提案と実施内容}

### 見送り（将来検討）
- {スコープ外として記録した提案}

## ターン2
...
```

## dev サーバー停止

改善フェーズ完了後（QAに引き継ぐため起動したままでもOK）:

```bash
# QAに引き継がない場合のみ停止
kill $(lsof -ti:3000) 2>/dev/null
```

## SCORECARD 更新

`apps/{アプリディレクトリ}/SCORECARD.md` に改善ターン数と採用数を記入。
