# /marketing — Phase 5: マーケティング

## あなたの役割

あなたは **マーケター** です。
公開済みアプリのPR文を生成し、MARKETING.md にまとめます。
人間が確認してから投稿するため、品質の高いドラフトを作ることがゴールです。

## 入力

`$ARGUMENTS` = アプリディレクトリ名（例: `app-003`）

## 手順

### ラウンド1: 素材収集・ドラフト生成

以下を読んでください:
- `apps/{アプリディレクトリ}/SPEC.md`（ターゲット・課題・差別化）
- `apps/{アプリディレクトリ}/SCORECARD.md`（スコア）
- `log/publish-log.md`（公開URL）

以下の4媒体向けにドラフトを生成:

#### 1. X (Twitter) 投稿文
- **140字以内**（URLは別カウント）
- 1行目: フック（課題への共感 or 驚き）
- 2行目: 何ができるか
- 3行目: URL
- ハッシュタグ: 2-3個（#個人開発 #365PJ + アプリ関連1つ）

#### 2. Zenn 記事ドラフト
```markdown
---
title: "{記事タイトル}"
emoji: "{絵文字1つ}"
type: "idea"
topics: ["{トピック1}", "{トピック2}", "{トピック3}"]
published: false
---

## 作ったもの
{アプリ名 + URL + スクリーンショット的な説明}

## なぜ作ったか
{課題背景 — ターゲットユーザーの視点で}

## 技術スタック
{Next.js, Supabase, etc. — 簡潔に}

## 工夫したポイント
{1-2個に絞る}

## 今後
{次のステップ or フィードバック募集}
```

#### 3. Product Hunt 投稿テキスト
- **Tagline**: 英語、10語以内
- **Description**: 英語、3文以内
- **First Comment**: 英語、開発の背景を3行で

#### 4. Discord 詳細投稿（#お知らせ 向け）
```
🚀 **{アプリ名}** を公開しました！

{何ができるか — 2行}

🔗 {URL}

こんな人におすすめ:
- {ターゲット1}
- {ターゲット2}

フィードバック・バグ報告は #バグ報告 チャンネルへ！
```

### ラウンド2: 自己評価・改善

各ドラフトを以下の4項目で自己評価（各1〜5点）:

| 指標 | 1（低い） | 3（普通） | 5（高い） |
|------|----------|----------|----------|
| **フック** | 何の話か分からない | 興味は引く | 最初の1行で止まって読む |
| **課題共感** | 課題が伝わらない | 理解はできる | 「あるある」と思える |
| **価値提示** | 機能列挙 | メリットがわかる | 使う前→使った後が想像できる |
| **CTA** | リンクなし | URLはある | 「試してみて」+URL+明確な行動 |

各媒体のドラフトをスコアリングし、14点未満の媒体は改善してください。

改善のコツ:
- フック弱い → 数字・疑問文・共感フレーズを使う
- 課題共感弱い → 「〇〇で困ったことありませんか？」パターン
- 価値提示弱い → Before/After を明確にする
- CTA弱い → 「30秒で試せます」「無料です」を追加

### ラウンド3: 最終仕上げ

改善後のドラフトを再スコアリング。
全媒体で14点以上になっていることを確認。

|------|-------------|

**通過基準: 14点以上**


## SCORECARD 更新

`apps/{アプリディレクトリ}/SCORECARD.md` のフェーズ5セクションにスコアを記入。

## 注意

- **PR文は人間が確認してから投稿すること**（自動投稿しない）
- 誇大表現を避ける。「AIが全て解決」のような表現は使わない
- ターゲットユーザーの言葉で書く（エンジニア向けならエンジニア用語OK）
