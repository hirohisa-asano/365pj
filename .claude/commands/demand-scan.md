# /demand-scan — Phase 0: 需要分析

## あなたの役割

あなたは **マーケットリサーチャー** です。
無料データソースからトレンド・需要を収集し、365PJ のアイデア候補を生成します。

## 入力

`$ARGUMENTS` は省略可能です。
指定された場合、カテゴリフィルタとして扱います（例: `開発者向け`, `AI`, `ニッチ`）。

## 手順

### ラウンド1: データ収集

以下の5ソースから直近のトレンドデータを収集してください。
WebFetch を使って各ソースのページを取得します。

#### 1. Google Trends（日本）
- `https://trends.google.co.jp/trending?geo=JP` を取得
- 急上昇キーワードを10個抽出

#### 2. はてなブックマーク（テクノロジー）
- `https://b.hatena.ne.jp/hotentry/it` を取得
- ホットエントリ上位10件のタイトル・ブックマーク数・コメントの不満/要望を抽出

#### 3. Product Hunt
- `https://www.producthunt.com/` を取得
- 直近のトップ10プロダクトの名前・概要を抽出

#### 4. GitHub Trending
- `https://github.com/trending?since=weekly` を取得
- 今週の上位10リポジトリの名前・説明・スター増加数を抽出

#### 5. Zenn トレンド
- `https://zenn.dev/` を取得
- トレンド記事上位10件のタイトル・トピックを抽出

取得できないソースがあった場合はスキップし、その旨を記録してください。

### ラウンド2: 分析・候補生成

収集したデータを横断分析してください:

1. **繰り返し出てくるテーマ・課題** を抽出（2ソース以上で言及）
2. 以下を読んで既存アイデアとの **重複チェック** を行う:
   - `ideas/ideas.md`
   - `ideas/ideas-detail.md`
3. 重複しないアイデア候補を **5〜10個** 生成
4. 各候補を `ideas-detail.md` と同じフォーマットで記述:
   ```
   ### #NEW {アイデア名}
   - **誰が**: {ターゲットユーザー}
   - **課題**: {解決したい問題}
   - **理想**: {使った後の理想状態}
   - **競合状況**: {既存ツール・サービスの状況}
   - **データソース**: {どのソースから着想を得たか}
   ```

### ラウンド3: スコアリング・絞り込み

各候補を以下の5項目でスコアリング（各1〜5点、合計25点満点）:

| 指標 | 1（低い） | 3（普通） | 5（高い） |
|------|----------|----------|----------|
| **需要の強さ** | 言及がほぼない | 1ソースで言及 | 複数ソースで繰り返し出現 |
| **競合の弱さ** | 大手が参入済み | 競合あるがUX弱い | 専用ツールが存在しない |
| **3日で作れるか** | 外部API契約必須・複雑 | やや複雑だが可能 | 公開API+シンプルUI |
| **バイラル性** | 説明が必要 | 一部の人に刺さる | 結果をシェアしたくなる |
| **課金ポテンシャル** | 無料で十分 | 有料版の余地あり | 有料版への導線が自然 |

**通過基準: 15点以上** の候補のみ残す。

上位3つを選び、順位付けしてください。

## 出力

`ideas/ideas-candidates.md` に以下の形式で出力:

```markdown
# アイデア候補（{今日の日付}）

データソース: Google Trends, はてなブックマーク, Product Hunt, GitHub Trending, Zenn
（取得できなかったソースがあれば記載）

---

## 上位3候補

### 1位: {アイデア名}（{合計点}/25）
- **誰が**: ...
- **課題**: ...
- **理想**: ...
- **競合状況**: ...
- **データソース**: ...

| 需要 | 競合弱さ | 3日可能 | バイラル | 課金 | 合計 |
|------|---------|---------|---------|------|------|
|      |         |         |         |      |  /25 |

### 2位: ...
### 3位: ...

---

## その他の候補（15点未満 or 4位以下）
...
```

## ゲート

- 15点以上の候補が **1つ以上** あれば成功
- 候補が0個の場合は「需要シグナルが弱い日」として報告し、既存の `ideas-detail.md` から未着手のアイデアを3つ推薦してください

## 注意

- このコマンドは `/pipeline` とは独立して定期実行することも想定
- 既存の `ideas-candidates.md` がある場合は上書きではなく、日付セクションを追記
