# IMPROVE — よしよしAI（Phase 2.7 プロダクト改善）

レビュー（設計/UI/セキュリティ）とローカル動作確認をもとに、以下を実装済み。

## 実装した改善

1. **配色バグ修正（最重要）**: `text-mutedForeground`（Tailwind v4 で生成されない無効クラス）を
   全17箇所 `text-muted-foreground` に統一。補足文・免責・ラベルが本来のやわらかいグレージュで
   表示されるようになり、アプリの生命線である「やわらかさ・安心感」を回復。
2. **ヒーローの世界観強化**: タイトル上にマスコット（🫂）＋やわらかいラジアルグラデの円を追加。
   文字だけだった第一印象に温かみと個性を付与。
3. **エラー表示をやさしく**: 赤字（text-red-500）をやめ、primary系のやわらかい角丸ボックスに変更。
   弱っている利用者に冷たく感じさせない。特に「1日3回」案内はエラーでなく導線として表示。
4. **結果画面に免責を追加**: 応援カード下に「医療目的でない／つらさが続くときは専門窓口へ」を明記
   （入力画面だけでなく結果画面でも安全メッセージが消えないように）。
5. **CTAにモーション**: 「よしよししてもらう」ボタンを PressScale で包み、押下時のバウンス感を追加。
6. **危機検知の強化**: ひらがな/カタカナ正規化・誤検知（"od"）修正（詳細は REVIEW-security.md）。
7. **プロンプトインジェクション対策**: 指示を system に分離、ユーザー入力をデータ扱い。
8. **dead code 削除**: 未使用の cardRef / useRef を除去。

## 見送り（今後の候補）

- サーバー側レート制限（#6）: 既存アプリ踏襲でクライアント制限のみ。将来バイラルで負荷が出たら
  Supabase または Vercel KV で厳格化を検討（人間判断で今回は許容）。
- 応援メッセージの音声読み上げ（SpeechSynthesis）: 看護師トーンと相性が良さそう。
- 応援ログの画像まとめ（週次「がんばり通知簿」）: normal プランの付加価値として。
