# よしよしAI — SPEC

## 概要

今日ダメだったこと・できなかったことを打ち込むと、AIが全力で肯定・応援してくれるアプリ。
成果や行動は一切求めず、ただ「よしよし」して味方でいてくれる。応援の熱量（優しめ⇔強め）と
「誰に応援されたいか」（人格プリセット）を自分で選べるのが核。

## ターゲットユーザー

- 自己肯定感が低く、自分を責めがちな人（学生〜社会人）
- うつ気味・就活疲れ・育児/介護疲れ・燃え尽きで「頑張れない自分」を責めている人
- カウンセリングはハードルが高いが、気軽に褒めてくれる相手が欲しい人

## 課題

- 「頑張れない自分」を責め続ける負のループから抜け出せない
- 周りに「そのままでいい」と言ってくれる人がいない
- SNSは他人の成功で溢れ、比較して余計に落ち込む
- 既存のメンタルアプリは「記録・改善」志向で、ただ肯定してくれるものが少ない

## 理想の体験

「今日も二度寝して何もできなかった」と打ち込む → 選んだ人格・トーンで、AIが本気で拾い上げて
肯定してくれる。「そんな見方があったのか」と少し心が軽くなる。もらった言葉を保存・シェアして
また元気を補充したくなる。

## 競合状況

- **muute / emol など感情記録アプリ**: 記録・分析が主。肯定に全振りではない
- **ChatGPT等**: 汎用。人格・トーン調整の専用UXがなく、毎回プロンプトを書く必要がある
- **差別化**: 「行動を一切求めない・肯定に全振り」＋「トーン5段階×人格プリセット」の専用体験。
  もらった応援を画像化してシェアできるポジティブなバイラル導線

## 画面構成

- `/` — 1画面完結（CSRインタラクション）
  - ヘッドライン + サブコピー + 医療目的でない旨の一言
  - **今日の調子（気力メーター）**: げんき / ふつう / しんどい / げきしんどい を選択。看護師の問診のように、
    まず今日のコンディションを尋ねる。低い日は自動で優しめトーンに寄せる
  - **人格プリセット選択**（横スクロールのチップ）: 推し（王道）/ 推し（ツンデレ）/ 推し（萌え萌え）/ 体育会系コーチ / おじいちゃん
    - 体育会系コーチは叱咤・喝を許可（「そんなんじゃダメだろ」等）。ただし存在価値の否定は禁止＝愛のある喝
  - **ニックネーム入力（全プラン共通・任意）**: 呼んでほしい名前を設定すると、推し等がその名前で呼んで応援する
  - **応援トーンスライダー**: 優しめ ⇔ 強め（5段階）
  - テキスト入力（今日のダメ報告。プレースホルダーで例示）
  - 「よしよししてもらう」ボタン + 「迷ったらこれで試す」サンプル投入
  - ローディング演出
  - 結果: 大きめの「応援カード」（人格アイコン + 応援メッセージ）
  - アクション: 画像で保存/シェア / コピー / もう一度
  - （ログイン時）応援ログ: 過去にもらった言葉を端末に保存して見返せる
  - 危機検知時: 応援を出さず、相談窓口カード（いのちの電話等）を表示

## 技術仕様

- レンダリング: SSG（インタラクションは CSR）
- DB: 全アプリ共通の usage_counts（回数制限のサーバー集計）のみ使用。応援ログは localStorage、認証は Supabase Auth
- 認証: 全アプリ共通の AuthHeader（テンプレートから継承）
- 外部API: Claude API（応援メッセージ生成、`claude-haiku-4-5-20251001`）
- 画像生成: Canvas API（Tier 0）で応援カードを描画してダウンロード/シェア
- 使用サービス（Tier別）:
  - Tier 0: Canvas API（画像化）, Web Share API, Clipboard API, localStorage
  - Tier 1: Claude API, Supabase（認証のみ）
  - Tier 2: なし

## 安全設計（最重要）

- API側で希死念慮・自傷を示すワードを検知したら、応援メッセージを生成せず
  「相談窓口カード」（いのちの電話 0120-783-556 / よりそいホットライン 0120-279-338 等）を返す
- 画面下部と結果に「これは医療・治療を目的としたサービスではありません」の免責を明記
- 強めトーンでも人格否定・暴言・皮肉は禁止（システムプロンプトで厳守。あくまで肯定ベースの熱い応援）
- プライバシー: 入力・応援ログは端末（localStorage）のみ。サーバーに保存しない

## プラン設計

| 機能 | minimum（未ログイン） | medium（Google認証） | normal（サポーター） |
|------|---------------------|--------------------|--------------------|
| 応援してもらう回数 | 1日3回（端末） | 1日10回（サーバー集計） | 1日50回（サーバー集計） |
| トーン調整 | 2段階（優しめ/強め） | 5段階フル | 5段階フル |
| 人格プリセット | 1種（推し王道） | 全5種 | 全5種 |
| ニックネーム指定 | ○ | ○ | ○ |
| 画像でシェア | ○ | ○ | ○ |
| 応援ログ保存（端末） | × | ○ | ○ |
| カスタム推し（自作プロンプト） | × | ○（120字まで） | ○（500字まで） |

## 機能要件

1. 人格プリセットを選択できる（推し王道/推しツンデレ/推し萌え萌え/体育会系コーチ/おじいちゃん。プラン別に選択可能数を制限）
2. トーンをスライダーで調整できる（プラン別に段階数を制限）
3. 今日の調子（気力メーター）を選び、ダメ報告を入力して、選んだ人格・トーン・調子で応援メッセージを生成する（調子が低い日は優しめに寄せる）
4. 回数制限: 未ログインは1日3回（localStorage）、ログインは1日10回、サポーターは1日50回（ログイン以上はSupabase usage_countsでサーバー側集計＝バイパス不可）
5. 危機ワード検知時は応援を出さず相談窓口を表示する
6. 応援メッセージを画像化して保存/シェアできる（Canvas）
7. 応援メッセージをクリップボードにコピーできる
8. ログイン時、もらった応援を端末に保存し見返せる（応援ログ）
9. カスタム推し（自作プロンプト）を作れる。ログインユーザーは120字まで、サポーターは500字まで（未ログインは不可・サーバー側で会員検証）
10. 医療目的でない旨の免責を表示する

## スコープ外

- サーバー側でのデータ永続化（DB）／アカウント間同期
- 通知・リマインド機能
- コミュニティ・他ユーザーとの共有ギャラリー
- 音声・画像入力
