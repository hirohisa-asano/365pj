# BucketAI — SPEC

## 概要

3つの質問に答えるだけで、AIがあなただけの「死ぬまでにやりたいことリスト」を10個生成する。

## ターゲットユーザー

- 「何かやりたいけど、何がやりたいかわからない」人
- 年末・誕生日・転機に人生を振り返りたい人
- バケットリストを作りたいが白紙の前で固まる人

## 課題

- バケットリストアプリは「自分で書く」前提。書けない人には役に立たない
- ChatGPTでも聞けるが「対話で引き出す」専用体験がない
- 既存アプリは管理ツール。「発見・気づき」の体験が弱い

## 理想の体験

3つの質問（価値観・好き・後悔）に答えたら、自分では思いつかなかった「やりたいこと」が10個出てくる。「これ、確かにやりたい」という発見がある。リストをシェアしたくなる。

## 競合状況

- **LIST**: やりたいことリスト管理。サービス終了
- **MY BUCKET LIST 100**: リスト管理+画像。自分で書く前提
- **カナエル**: 残り時間カウントダウン付き。リスト管理
- **差別化**: 「自分で書く」ではなく「AIが引き出す」。質問に答えるだけ

## 画面構成

- `/` — 1画面完結
  - ヘッドライン + サブコピー
  - ステップ形式の3つの質問（1問ずつ表示、回答するたび次の質問へ）
    - Q1: 「お金も時間も無限にあったら、明日何をする？」（自由回答）
    - Q2: 「人生で一番楽しかった瞬間は？」（自由回答）
    - Q3: 「死ぬ時に『あれやっておけばよかった』と思いそうなことは？」（自由回答）
  - 「リストを生成する」ボタン
  - 結果: 10個のバケットリスト（番号 + タイトル + なぜあなたに合うかの一言理由）
  - シェアボタン + もう一度やるボタン

## 技術仕様

- レンダリング: SSG（インタラクションは CSR）
- DB: 使用しない
- 認証: 全アプリ共通の AuthHeader（テンプレートから継承）
- 外部API: Claude API（バケットリスト生成）
- 使用サービス（Tier別）:
  - Tier 0: なし
  - Tier 1: Claude API, Supabase（認証のみ）
  - Tier 2: なし

## プラン設計

| 機能 | minimum（未ログイン） | medium（Google認証） | normal（サポーター） |
|------|---------------------|--------------------|--------------------|
| リスト生成 | 1日1回 | 1日5回 | 無制限 |
| 結果表示 | 10個中5個のみ表示 | 全10個表示 | 全10個表示 |
| シェア | OK | OK | OK |

※ minimum の回数制限は localStorage で管理

## 機能要件

1. ステップ形式で3つの質問を1問ずつ表示（回答→次の質問へアニメーション遷移）
2. 全質問回答後「リストを生成する」ボタンが表示される
3. Claude API にユーザーの回答を送り、バケットリスト10個を生成
4. 結果は番号 + タイトル + 一言理由の形式で表示（Staggerアニメーション）
5. minimum プランでは10個中5個のみ表示し、残り5個は「ログインすると全て見れます」
6. ローディング中は「あなたの人生を読み解いています...」の演出表示
7. 「結果をシェアする」ボタン（Web Share API / X投稿フォールバック）
8. 「もう一度やる」ボタンで質問に戻る
9. レスポンシブ対応（モバイルファースト）
10. OGPにアプリ名・説明を設定

## デザインの方向性

### コンセプト

「星空の下で人生を語る」— 夜空のような深いダークブルーに、星のようなアクセント。内省的で温かい雰囲気。

### リファレンスサイト
- LIST (100list.app): 青×白のクリーンなデザイン。ナンバリング(#01〜#04)の使い方
- カナエル: 残り時間の見せ方。人生の有限性を意識させるUI

### リファレンスから学んだこと
- レイアウト: 1カラム中央寄せ。質問→結果の縦の流れ
- 色使い: ダーク背景 × 温かいアクセント（ゴールド系）
- 余白: 質問ごとにゆったりした余白。急かさない
- 個性: ステップ形式の質問が「対話感」を出す

### 避けるべきパターン
- チェックリスト的な事務的デザイン
- カラフルすぎるポップな配色（内省の邪魔）
- 情報密度の高いレイアウト（ゆったりさが大事）

### UI の個性
- 質問画面は大きな文字で1問だけ表示。余計な情報なし
- 回答テキストエリアは罫線なし、プレースホルダーが語りかけるように
- 結果の各項目に小さなアイコンや絵文字（🌍✈️🎸 等）を添えて視覚的な楽しさ

### モーション計画
- ページ読み込み: FadeIn で全体表示
- 質問遷移: FadeIn direction="left" で次の質問がスライドイン
- 結果表示: Stagger で10個が順番に現れる
- 操作フィードバック: PressScale でボタン
- ローディング: animate-pulse-slow で「読み解いています...」

## スコープ外

- リストの保存・編集機能
- 他ユーザーのリスト閲覧
- リストのPDF/画像エクスポート
- リマインダー機能
