# FilmPick — SPEC

## 概要

好きな映画・今の気分・観たい雰囲気を伝えると、あなたの趣味傾向を分析して「次に観るべき映画」を提案してくれるAIツール。

## ターゲットユーザー

- 次に観る映画を探している映画好き（ライト〜ミドル層）
- 「あの感じの映画がまた観たい」を言語化できない人
- ランキングや口コミではなく、自分の好みに合った作品に出会いたい人

## 課題

- ランキングは「みんなが観る映画」で自分好みとは限らない
- SNSの口コミは散漫で探しにくい
- 「あの雰囲気の映画がもっと観たい」を検索できない
- Filmarksは記録・レビュー特化で「探す体験」が弱い

## 理想の体験

好きな映画を2〜3本と今の気分を伝えたら、趣味の傾向を言語化してくれて、知らなかった作品に出会える。「なぜこの映画があなたに合うのか」の理由付きで、納得感がある。

## 競合状況

- **Filmarks**: レビュー・記録特化。レコメンドはランキングベース
- **TasteDive**: 英語のみ。「この映画が好きなら→これ」のシンプルな形式
- **taste.io**: 協調フィルタリングベース。日本語非対応
- **差別化**: 日本語対応 × 気分・雰囲気入力 × 趣味傾向の言語化 × 理由付き推薦

## 画面構成

- `/` — トップページ（1画面完結）
  - ヘッドライン + サブコピー
  - 入力エリア: 好きな映画（テキスト入力、2〜3本）+ 気分セレクト（チップ選択式）
  - 「映画を探す」ボタン
  - 結果エリア: 趣味分析 + おすすめ映画3本（タイトル・ジャンル・年代・おすすめ理由）
  - 「もう一度探す」で入力に戻る

## 技術仕様

- レンダリング: SSG（結果表示は CSR）
- DB: 使用しない
- 認証: 不要
- 外部API: Claude API（映画推薦 + 趣味分析）
- 使用サービス（Tier別）:
  - Tier 0: なし
  - Tier 1: Claude API
  - Tier 2: なし

## 機能要件

1. 好きな映画を2〜3本テキスト入力できる（入力フィールド × 3、最低1本で送信可能）
2. 今の気分をチップ選択できる（「じっくり考えたい」「スカッとしたい」「泣きたい」「ワクワクしたい」「怖い思いしたい」「笑いたい」「感動したい」「何も考えたくない」から複数選択）
3. 送信するとClaude APIに好きな映画 + 気分を送り、以下を生成:
   - 趣味傾向の分析テキスト（「あなたは○○な作品が好きな傾向があります」）
   - おすすめ映画3本（タイトル / 監督 / 年 / ジャンル / おすすめ理由）
4. 結果はアニメーション付きで表示（趣味分析 → 映画が順番に出てくる）
5. 「もう一度探す」ボタンで入力状態にリセットできる
6. ローディング中は「あなたの好みを分析中...」の演出表示
7. レスポンシブ対応（モバイルファースト）
8. OGPにアプリ名・説明を設定

## デザインの方向性

### コンセプト
「映画館の暗がりで友達に相談している感じ」— 暗めの背景に映画のポスターのような色使い。

### リファレンスサイト
- TasteDive: https://tastedive.com/movies — 検索中心のレイアウト、ダークネイビー×オレンジのアクセント
- Filmarks: https://filmarks.com/ — 情報密度が高い日本的レイアウト（こうならない方向で参考）

### リファレンスから学んだこと
- レイアウト: 1カラム中心。入力→結果の縦の流れ
- 色使い: ダーク背景 × 暖色アクセント（映画館の雰囲気）
- 余白: 入力エリアは余裕を持たせ、結果エリアは密度を上げる
- 個性: 気分チップの選択体験が楽しさのフック

### 避けるべきパターン
- 映画ポスター画像を並べるだけ（画像APIがないので文字で勝負）
- 白背景のフォーム感（映画の雰囲気が出ない）
- ランキングリスト感（既存サービスと同じに見える）

### UI の個性
- ダーク背景（#0F0F1A 系）にグラデーションのアクセント（アンバー→コーラル）
- 気分チップは映画のチケットスタブ風のデザイン
- 結果の映画カードは「映画のワンシーン」を想起させるグラデーション背景 + 大きなタイトル表示

### モーション計画
- ページ読み込み: FadeIn で全体表示
- 結果表示: Stagger で趣味分析 → 映画1 → 映画2 → 映画3 と順番に
- 操作フィードバック: PressScale でチップ選択・ボタン
- ローディング: animate-pulse-slow で「分析中...」テキストが脈動

## スコープ外

- ユーザー登録・ログイン
- 映画のポスター画像表示（外部画像APIが必要なため）
- 映画の視聴リンク（配信プラットフォームへの導線）
- 履歴保存
- 映画評価・レビュー機能
