# 365PJ — 2026年 個人開発チャレンジ

## コンセプト

アイデアを大量に出し、小さく素早く作り、市場の反応を見る。

大作を1つ作るのではなく、多数の実験を行い、反応のあったものだけを育てる。

---

## フェーズ

### Phase 1: アイデア出し
**目標：365個**

- 良し悪しを判断しない
- 収益性も考えない
- とにかく数を出す
- 自分の経験や興味を活用する

**対象領域例**
エンジニアリング / AI / PBX / 投資 / 哲学 / SCU / ピアノ / 宇宙 / 日常の不便解決

→ [`ideas/ideas.md`](ideas/ideas.md)

---

### Phase 2: プロトタイプ制作
**目標：10個**

**選定基準**
- 数日以内で作れる
- 単機能
- 小規模
- まずは無料公開

作り込みすぎない。

→ [`apps/`](apps/)

---

### Phase 3: 公開
**開始予定：2026年7月12日（誕生日）**

- 1日1個ずつ公開
- 反応を観測
- SNSやブログで発信
- 完璧を求めない

→ [`log/`](log/)

---

### Phase 4: 育成

以下のような反応があったものを優先して育てる。

- 利用者が継続利用している
- 問い合わせが来る
- シェアされる
- SEO流入がある
- 自分自身が使い続けている

反応がないものは潔く放置する。

---

## 制約条件

### 開発環境
- **Replit** をメインに使う（ローカル環境構築不要・即公開できる）

### 1本あたりの制約
| 項目 | 制約 |
| -- | -- |
| 開発期間 | 最大3日以内 |
| 機能数 | 単機能（1アプリ1目的） |
| 規模 | 小規模（数百行以内を目安） |
| 公開方法 | まず無料公開 |
| 作り込み | しない。動けばOK |

### 寄付動線
- 各アプリに **Buy Me a Coffee** または **Ko-fi** のリンクを置く
- 強制しない・目立たせすぎない
- 「気に入ったら」程度のトーンで添える
- 寄付があったアプリ ＝ 育成候補の有力シグナル
