# アイデア詳細設計書

> 20個のアイデアを最終精査し、**作る価値がある順**に詳細設計まで落としたもの。
> 評価基準：3日以内に作れるか / 競合との差別化 / 収益ポテンシャル / バイラル性 / 技術的実現性

## 最終精査：20 → 10

### 除外した10個と理由

| アイデア | 除外理由 |
| -- | -- |
| **コードレビューAI** | GitHub連携が必須で3日では厳しい。CodeRabbit/Copilotも進化中 |
| **アーキテクチャレビュー** | 入力（設計図・要件）が複雑すぎる。UXの設計だけで3日かかる |
| **エラーログ分析** | Sentry/Datadogが無料枠で十分。シンプル版では価値が薄い |
| **テストケース生成** | コード入力UIが重い。Copilotの進化で価値が目減り中 |
| **補助金検索** | データの鮮度・正確性がAIだけでは担保できない。誤情報リスクが高い |
| **YouTubeチャンネル探索AI** | YouTube API連携が必要。API制限・コストが読めない |
| **オタクコレクション管理** | CRUD画面が多く3日では中途半端になる。バーコード連携も重い |
| **ガジェット比較・提案AI** | 最新製品データの鮮度が命。AIの知識だけでは古い情報になる |
| **スポーツ観戦提案AI** | ターゲットが狭すぎる。日本のスポーツ市場でWebアプリに課金動機が弱い |
| **芸人提案AI** | ターゲットが狭い。漫画AIと同パターンで新規性がない |

### 残った10個（優先順位順）

| 順位 | アイデア | 決め手 |
| -- | -- | -- |

| 7 | **ソクラテスと対話** | 唯一無二の体験。哲学好きのコア層に深く刺さる |
| 8 | **AI壁打ち** | 思考整理に特化したUXで差別化 |
| 9 | **LLMプロンプト最適化** | AI開発者向け。改善提案のUXが作れる |

### #1 LLMコスト計算（app-002）

**サービス名案**: LLM Pricing Lab / トークン電卓

#### コンセプト
AI開発者が「このモデルで月いくらかかるか」を30秒で把握できるツール。

#### ターゲット
- AI機能を組み込んでいるエンジニア
- PdM（プロダクトマネージャー）がコスト承認する際の根拠資料として

#### 課題の深さ
- 各社の料金ページがバラバラで比較できない
- トークン数 → 月額の換算が面倒
- モデル変更時のコスト差が一目でわからない

#### 画面構成
```
/ （1ページ完結）
├── 入力エリア
│   ├── 想定リクエスト数/日（スライダー or 数値入力）
│   ├── 平均入力トークン数（プリセット: 短文500 / 中文2000 / 長文8000）
│   ├── 平均出力トークン数（プリセット: 短文200 / 中文1000 / 長文4000）
│   └── キャッシュ率（0〜100%）
├── 結果エリア
│   ├── モデル比較テーブル（GPT-4o / Claude Sonnet / Gemini Pro 等）
│   │   └── 各行: モデル名 / 入力単価 / 出力単価 / 月額見積 / 年額見積
│   ├── コスト棒グラフ（視覚比較）
│   └── 「このモデルなら月○○円」のハイライト
└── フッター（寄付 + ポータルリンク）
```

#### 技術仕様
- **レンダリング**: SSG（全てクライアントサイド計算）
- **DB**: 不要
- **認証**: 不要
- **外部API**: 不要（料金データはハードコード → 定期的に手動更新）
- **特徴**: AI API を一切使わない純粋な計算ツール。コストゼロ運用。

#### データ構造（料金マスタ）
```ts
type Model = {
  provider: string        // "OpenAI" | "Anthropic" | "Google"
  name: string            // "GPT-4o"
  inputPer1M: number      // 入力100万トークンあたりの価格（USD）
  outputPer1M: number     // 出力100万トークンあたりの価格（USD）
  cachedInputPer1M?: number // キャッシュヒット時の入力価格
  contextWindow: number   // コンテキストウィンドウ
  updatedAt: string       // 料金更新日
}
```

#### トーンマナー
- **ダーク系**: 開発者向けなのでダークモード基調
- **アクセント**: グリーン（お金 → 節約のイメージ）
- **フォント**: モノスペース寄り（数字が多いため）

#### 開発見積
| Day | 内容 |
| -- | -- |
| 1 | 料金データ整備 + 計算ロジック + 基本UI |
| 2 | グラフ表示 + レスポンシブ + プリセット |
| 3 | OGP + デプロイ + 微調整 |

#### 収益モデル
- 寄付ベース（初期）
- 将来: API使用量の自動取得（有料機能）、チーム向けコスト管理ダッシュボード

### #2 転職偏差値（app-003）

**サービス名案**: キャリア偏差値 / 転職スカウター

#### コンセプト
経歴・スキル・年収を入力すると「転職偏差値」が数値で出る。SNSでシェアしたくなるバイラル設計。

#### ターゲット
- 転職を検討中の20〜40代ビジネスパーソン
- 自分の市場価値を知りたい人（転職意思がなくても）

#### 課題の深さ
- 自分の市場価値がわからない
- ミイダスは「年収診断」で偏差値形式ではない
- シェアしたくなる形式のものがない

#### 画面構成
```
/ （入力ページ）
├── ステップ形式の入力フォーム（3ステップ）
│   ├── Step 1: 基本情報（年齢・職種・業種）
│   ├── Step 2: スキル・経験（年数・資格・マネジメント経験）
│   └── Step 3: 現在の年収
└── 送信ボタン

/result （結果ページ）
├── 偏差値（大きな数字で表示: 例「62.3」）
├── ランク表示（S/A/B/C/D + 説明）
├── レーダーチャート（スキル・経験・年収・将来性・希少性）
├── 「狙える企業ランク」の目安
├── 改善アドバイス（3つ）
├── SNSシェアボタン（X / LINE）
│   └── 「私の転職偏差値は62.3でした #転職偏差値」
└── フッター
```

#### 技術仕様
- **レンダリング**: SSG + クライアントサイドAPI呼び出し
- **DB**: 不要（結果は保存しない。シンプルに保つ）
- **認証**: 不要
- **外部API**: Claude API（偏差値算出 + アドバイス生成）
- **OGP**: 動的生成（偏差値を含むOGP画像 → シェア時に数字が見える）

#### AIプロンプト設計
```
あなたは転職市場の専門家です。以下の情報から「転職偏差値」を算出してください。

【入力】
- 年齢: {age}
- 職種: {job}
- 業種: {industry}
- 経験年数: {years}
- スキル: {skills}
- マネジメント経験: {management}
- 現在の年収: {salary}万円

【出力（JSON）】
{
  "score": 62.3,
  "rank": "A",
  "radar": { "skill": 70, "experience": 65, "salary": 55, "potential": 75, "rarity": 60 },
  "targetCompanyRank": "大手〜メガベンチャー",
  "advice": ["...", "...", "..."]
}
```

#### バイラル設計（重要）
- 偏差値という「数字」がシェアの動機になる
- OGP画像に偏差値を埋め込み、Xでシェア時に目を引く
- 結果ページのURLにパラメータを含めず（プライバシー配慮）、OGPは汎用
- 「友達にも診断してもらう」導線

#### トーンマナー
- **クリーン系**: 白背景 + ブルー系アクセント（信頼感）
- **数字を大きく**: 偏差値が主役

#### 開発見積
| Day | 内容 |
| -- | -- |
| 1 | ステップフォーム + API連携 + プロンプト調整 |
| 2 | 結果表示UI + レーダーチャート + シェアボタン |
| 3 | OGP + デプロイ + 微調整 |

#### 収益モデル
- 寄付ベース（初期）
- 将来: 詳細レポート（有料）、転職エージェントアフィリエイト

### #3 炎上リスク診断（app-004）

**サービス名案**: 炎上チェッカー / FireCheck

#### コンセプト
SNS投稿前にテキストを貼り付けると、炎上リスクをスコア + 理由 + 修正案で返すツール。

#### ターゲット
- 企業のSNS担当・PR担当
- インフルエンサー・個人で発信力がある人
- 広報部門（B2B）

#### 課題の深さ
- 投稿してから炎上に気づく（取り返しがつかない）
- 社内チェックは属人的で基準がない
- 「このくらい大丈夫だろう」の判断ミスが致命的

#### 画面構成
```
/ （1ページ完結）
├── 入力エリア
│   ├── テキストエリア（投稿文を貼り付け）
│   ├── プラットフォーム選択（X / Instagram / Facebook / プレスリリース）
│   └── 業種選択（任意。業種固有のリスクを考慮）
├── 結果エリア
│   ├── リスクレベル（🟢安全 / 🟡注意 / 🔴危険）+ スコア（0-100）
│   ├── リスク要因リスト（箇条書き + 該当箇所ハイライト）
│   │   例: 「特定の属性への偏見と受け取られる可能性」
│   ├── 修正案（AIが書き換えた安全版を提示）
│   └── 「チェック済み」バッジ画像DL（社内承認フローに使える）
└── フッター
```

#### 技術仕様
- **レンダリング**: SSG + クライアントサイドAPI呼び出し
- **DB**: 不要（投稿文を保存しない → プライバシー配慮）
- **認証**: 不要
- **外部API**: Claude API

#### AIプロンプト設計
```
あなたはSNS炎上リスクの専門家です。以下の投稿文を分析し、炎上リスクを評価してください。

【プラットフォーム】{platform}
【業種】{industry}
【投稿文】
{text}

【評価観点】
- ジェンダー・人種・年齢等への偏見
- 政治・宗教への不用意な言及
- 誇大表現・景品表示法違反の可能性
- 著作権・肖像権リスク
- 業種固有のコンプライアンスリスク
- 文脈なしで切り取られた場合の誤解リスク

【出力（JSON）】
{
  "score": 72,
  "level": "warning",
  "risks": [
    { "type": "ジェンダー", "detail": "...", "highlight": "該当箇所" }
  ],
  "revision": "修正後の投稿文",
  "summary": "一言まとめ"
}
```

#### トーンマナー
- **プロフェッショナル系**: 白背景 + レッド/イエロー/グリーンの信号色
- **信頼感**: 法務・コンプライアンスの雰囲気

#### 開発見積
| Day | 内容 |
| -- | -- |
| 1 | 入力フォーム + API連携 + プロンプト調整 |
| 2 | 結果表示UI + リスクハイライト + 修正案表示 |

### #4 先延ばし検知AI（app-005）

**サービス名案**: 先延ばしラボ / Procrastination Detective

#### コンセプト
「なぜ先延ばしているのか」を対話で分析し、原因パターンと具体的な打ち手を返すツール。

#### ターゲット
- ADHD傾向のある人
- 生産性に悩むビジネスパーソン
- 学生（レポート・試験勉強の先延ばし）

#### 画面構成
```
/ （1ページ完結）
├── 入力エリア
│   ├── 「今、先延ばしていることは？」（テキスト入力）
│   ├── 「いつまでにやる必要がある？」（日付 or 選択: 今日 / 明日 / 今週 / 来週）
│   └── 「なぜ手をつけられない？」（タグ選択: 面倒 / 不安 / 完璧主義 / やり方不明 / やる気が出ない）
├── 結果エリア
│   ├── 先延ばしタイプ（4象限で分類）
│   │   ├── 回避型（不安・恐怖が原因）
│   │   ├── 決断麻痺型（選択肢が多すぎる）
│   │   ├── 完璧主義型（100%じゃないとやれない）
│   │   └── 刺激不足型（つまらないからやれない）
│   ├── 原因分析（なぜこのタスクを先延ばしているか）
│   ├── 打ち手（3つの具体的アクション）
│   │   例: 「まず2分だけやる」「完成度30%で一旦出す」「場所を変える」
│   └── 「5分後にもう一度チェックする」リマインダー設定（任意）
└── フッター
```

#### 技術仕様
- **レンダリング**: SSG + クライアントサイドAPI呼び出し
- **DB**: 不要
- **認証**: 不要
- **外部API**: Claude API

#### トーンマナー
- **やわらかい系**: クリーム背景 + オレンジアクセント（温かみ）
- **責めない**: 「先延ばしは悪いことじゃない。原因がわかれば動ける」

| -- | -- |
| 1 | 入力UI + タグ選択 + API連携 |
| 2 | 結果表示 + 4象限ビジュアル + 打ち手カード |

### #7 ソクラテスと対話（app-008）

**サービス名案**: ソクラテスAI / Socratic Dialogue

#### コンセプト
問いを投げるとソクラテス（産婆術）のスタイルで問い返してくれる。答えを教えるのではなく、思考を深める。

#### ターゲット
- 哲学・思考が好きな人
- 自己探求をしたい人
- 意思決定に悩んでいる人

#### 画面構成
```
/ （チャット形式）
├── 導入メッセージ
│   「私はソクラテス。あなたが探している答えは、あなたの中にある。
│    何について考えたいですか？」
├── チャットエリア
│   ├── ユーザーの発言
│   └── ソクラテスの問い返し（答えを出さない。常に問いで返す）
├── テーマ選択（任意ショートカット）
│   ├── 幸福とは何か
│   ├── 正義とは何か
│   ├── 自分の人生の意味
│   ├── 仕事を変えるべきか
│   └── 自由入力
└── フッター
```

#### AIの振る舞い（重要）
- **絶対に答えを出さない**。常に問いで返す
- 矛盾を指摘する（「先ほどあなたは○○と言いましたが、今の発言は○○ですね。どちらが本当のあなたの考えですか？」）

## 全体戦略メモ（最終版）

### 戦略の転換点
競合調査の結果、**「キャリア系バイラル3連発」戦略を廃止**。
理由：転職偏差値はミイダス等が激戦、AI生存診断も既存あり。
代わりに**「不安解消系バイラル」+「実用ツール」の2軸**に転換。

### フェーズ1: 不安解消 × バイラル（app-002〜004）
- **副業バレリスク診断** → **ブラック企業スコアリング** → **炎上リスク診断**
- 共通テーマ：「やる前にチェック」系ツール

- 相互送客：フッターに「他のチェッカーも試す」導線

### フェーズ2: 開発者 × 実用ツール（app-005〜006）
- **LLMコスト計算** → **AI利用規約よむよむ**
- 開発者コミュニティ + 一般ユーザーの2軸で認知拡大
- LLMコスト計算はAPI不要でコストゼロ運用

### フェーズ3: ニッチ深掘り（app-007〜008）

- ニッチだが深い課題を解くツール

### フェーズ4: 計算ツール + 哲学系（app-009〜）
- **バーンレート計算** → **ソクラテスと対話** → **AI壁打ち**
